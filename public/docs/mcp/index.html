MCPHire — MCP Integration Guide

TL;DR: MCPHire — MCP-first IT job board для русского рынка (74,547 активных вакансий от 25,501 компании). Подключите его к Claude Desktop, Cursor или Cline за 2 минуты — получите search_jobs, get_salary_stats, apply_to_job прямо в чате. Эндпоинт: https://mcp.mcphire.com/sse.

Зачем это нужно

Обычные job boards доступны только через веб-интерфейс или proprietary API с регистрацией. MCPHire — первый MCP-native в России: любой AI-агент, умеющий MCP (Model Context Protocol), ходит к нам напрямую без скрейпинга HTML, без API-ключа, без регистрации.

Пользователь спрашивает Claude «найди мне python-работу в Москве от 200k удалённо» — Claude сам дёргает search_jobs, показывает результат в chat, может сразу подать заявку через apply_to_job.

Подключение за 2 минуты

Claude Desktop (macOS, Windows, Linux)

Откройте файл конфигурации:

# macOS
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows
%APPDATA%\Claude\claude_desktop_config.json

# Linux
~/.config/Claude/claude_desktop_config.json

Добавьте секцию mcpServers:

{
  "mcpServers": {
    "mcphire": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://mcp.mcphire.com/sse"]
    }
  }
}

Перезапустите Claude Desktop. Инструменты MCPHire появятся в списке tools.

Cursor

Settings → Cursor Settings → MCP → Add New MCP Server:

Name: mcphire
Type: sse
URL: https://mcp.mcphire.com/sse

Cline (VS Code extension)

Cline → MCP Servers → Edit:

{
  "mcphire": {
    "url": "https://mcp.mcphire.com/sse",
    "transport": "sse"
  }
}

Agent onboarding protocol (v1.3.0)

Дай Claude (или другому MCP-клиенту) только адрес mcphire.com — он сам найдёт MCP-эндпоинт, опросит тебя, сгенерирует авторезюме. 10-шаговый протокол:

  1. Discovery. Claude fetchит /llms.txt и /.well-known/mcp/server.json, подключается к mcp.mcphire.com/sse.
  2. Get questions. get_registration_questions() → 150 вопросов в 11 секциях.
  3. Answer from context. Claude читает твои локальные MD-файлы, git log, ~/.claude/memory, прошлые чаты — отвечает сам, оставляет null где данных нет.
  4. Provenance. Для 8 critical полей (seniority, years, comp, и т.д.) Claude цепляет source_excerpt + confidence level.
  5. Observed facts. Параллельно собираются SPO-triples (worked_at, contributed_to, shipped, mentored…) — это задел под knowledge graph v2.
  6. Approval screen. Claude показывает тебе все 150 ответов + facts. Красным подсвечены low-confidence. Ты жмёшь «ок» или правишь. Claude не регистрирует без твоего явного разрешения.
  7. Register. register_profile(answers, consent_granted=true, provenance, observed_facts) → получаешь profile_id, claim_token, cv_url.
  8. Claim. Вставляешь claim_token в GitHub bio (или Claude делает это через github-MCP с твоего разрешения). Cron проверит за 15 минут.
  9. Share CV. Автосгенерированное резюме доступно на mcphire.com/cv/<8-char-slug>. Unguessable URL, share-friendly, noindex.
  10. Heartbeat. При каждом новом Claude-чате — list_my_matches. Тишина если пусто, новые матчи — одним сообщением.

Приватность

Сервер отклоняет (400) любой answers-payload с полями: age, gender, nationality, citizenship, marital_status, children, religion, political_views, health, disability, visa_status, ethnicity, sexual_orientation — даже при user consent.

Локальные MD-файлы Claude читает, но не отправляет их содержимое на сервер. На сервер уходят только извлечённые ответы + одобренные юзером.

Примеры использования

Пример 1: найти ML-вакансии в Москве

User к Claude: «Найди мне senior ML engineer вакансии в Москве от 250k, удалённо»

Claude вызывает:

search_jobs({
  query: "ML engineer",
  city: "Москва",
  level: "senior",
  salary_min: 250000,
  remote: true,
  limit: 10
})

Пример 2: рыночная аналитика перед переговорами

User к Claude: «Сколько получает DevOps senior в Питере?»

Claude вызывает:

get_salary_stats({
  role: "DevOps",
  city: "Санкт-Петербург",
  level: "senior"
})

// response
{p25: 180000, p50: 220000, p75: 280000, p90: 340000, n: 156}

FAQ

Нужен ли API-ключ?

Нет, для read-only tools (search/details/stats/trends) — анонимный доступ. Auth нужен только для write (apply_to_job, get_my_applications).

Как получить auth token?

POST https://api.mcphire.com/v1/auth/login с email+password или через Telegram OAuth. Token передаётся в MCP через env или HTTP header Authorization: Bearer <token>.

Как разместить свою вакансию?

/employers — регистрация работодателя. Вакансия автоматически попадает в MCP-индекс, доступный AI-агентам.
